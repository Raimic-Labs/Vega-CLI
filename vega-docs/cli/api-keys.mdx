# API Keys

Vega works with AI models from multiple providers. Each provider requires an API key. The good news: **most providers offer free tiers** that are more than enough to get started.

---

## Why API Keys?

Vega is a local CLI tool — it doesn't bundle its own AI infrastructure. Instead, it sends your prompts directly from your machine to the AI provider of your choice. This means:

- ✅ **Your data goes directly to the provider** — not through Raimic Labs servers
- ✅ **You control which provider you use**
- ✅ **Free tiers from most providers** are sufficient for daily use
- ✅ **Keys are stored locally** in `~/.vega/config.json` — never sent to us

## How to Get Each Key

### Google Gemini (Recommended — Free)

Google Gemini is Vega's default model provider and the easiest to get started with.

<Steps>
  <Step title="Go to Google AI Studio">
    Visit [aistudio.google.com](https://aistudio.google.com) and sign in with your Google account.
  </Step>
  <Step title="Get API Key">
    Click **"Get API key"** in the left sidebar. Then click **"Create API key"**.
  </Step>
  <Step title="Select a project">
    Select an existing Google Cloud project, or click **"Create API key in new project"** to create one automatically.
  </Step>
  <Step title="Copy the key">
    Copy the API key that appears. It looks like: `AIzaSy...`
  </Step>
  <Step title="Add to Vega">
    Inside Vega, type `/connect` and select **Google Gemini**, then paste your key.
  </Step>
</Steps>

<Note>
  The free tier for Gemini includes 15 requests per minute and up to 1 million tokens per day — plenty for active development use.
</Note>

## Config File Location

All API keys are stored in plain text (locally encrypted) in your home directory:

```
~/.vega/config.json
```

Example config structure:

```json
{
  "default_model": "gemini-2.5-pro",
  "default_agent": "auto",
  "api_keys": {
    "gemini": "AIzaSy...",
    "groq": "gsk_...",
    "deepseek": "sk-...",
    "openai": null,
    "anthropic": null,
    "nvidia": null
  }
}
```

<Warning>
  Never share your `~/.vega/config.json` file or commit it to version control. API keys grant full access to your provider accounts.
</Warning>

---

## Removing a Key

To remove an API key, edit the config file directly:

```bash
nano ~/.vega/config.json
```

Or run `/connect` and re-enter the provider — leave the field blank and press Enter to clear it.

---

## Key Security

- Keys are stored **only on your local machine**
- Vega makes API calls **directly to providers** — no proxy through Raimic Labs
- The `~/.vega/` directory is created with `700` permissions (owner-only access)
- We recommend adding `~/.vega/` to your `.gitignore` if you work in home-directory repos
