# API Keys

Vega works with AI models from multiple providers. Each provider requires an API key. The good news: **all four supported providers offer free tiers** that are more than enough to get started.

---

## Why API Keys?

Vega is a local CLI tool — it doesn't bundle its own AI infrastructure. Instead, it sends your prompts directly from your machine to the AI provider of your choice. This means:

- ✅ **Your data goes directly to the provider** — not through Raimic Labs servers
- ✅ **You control which provider you use**
- ✅ **Free tiers from all providers** are sufficient for daily use
- ✅ **Keys are stored locally** in `~/.vega/config.json` — never sent to us

## How to Get Each Key

### NVIDIA NIM (Recommended — Free)

NVIDIA NIM is Vega's default provider and powers the **CodeAgent** and **DebugAgent**.

<Steps>
  <Step title="Go to NVIDIA Build">
    Visit [build.nvidia.com](https://build.nvidia.com) and sign in with your NVIDIA account (or create one — it's free).
  </Step>
  <Step title="Get your API Key">
    Click **"Get API Key"** in the top-right corner. Copy the key — it starts with `nvapi-`.
  </Step>
  <Step title="Connect in Vega">
    Run `vega` and enter:
    ```
    /connect
    ```
    Select **NVIDIA NIM** and paste your key.
  </Step>
</Steps>

## Connecting via the Wizard

The easiest way to add a key is through the **first-run wizard** (launches automatically) or `/connect` inside a Vega session:

```bash
vega

# Inside Vega:
/connect
```

You'll see:

```
  Choose a provider:

    1. NVIDIA NIM (free tier)  →  https://build.nvidia.com
    2. Google Gemini           →  https://aistudio.google.com
    3. Groq (free tier)        →  https://console.groq.com
    4. DeepSeek                →  https://platform.deepseek.com
    5. Skip for now (set keys later with /connect)

  Choice [1-5]:
```

## Setting Keys via Environment Variables

You can set API keys without the wizard using environment variables. Vega checks these before reading `~/.vega/config.json`:

```bash
export VEGA_NVIDIA_API_KEY="nvapi-..."
export VEGA_GOOGLE_API_KEY="AIza..."
export VEGA_GROQ_API_KEY="gsk_..."
export VEGA_DEEPSEEK_API_KEY="sk-..."
```

Or use the generic form:

```bash
export NVIDIA_API_KEY="nvapi-..."
export GOOGLE_API_KEY="AIza..."
```

Add these to your `.bashrc`, `.zshrc`, or `.env` file to make them permanent.

## Managing Keys Directly

You can also view or update keys directly through the config:

```bash
# View current config (keys are shown as stored)
/settings

# Set a key manually
/config api_keys.nvidia nvapi-your-key-here
```

Or edit `~/.vega/config.json` directly:

```json
{
  "api_keys": {
    "nvidia":   "nvapi-...",
    "google":   "AIza...",
    "groq":     "gsk_...",
    "deepseek": "sk-..."
  }
}
```

## Switching Between Providers

Once you've added multiple keys, you can switch providers mid-session:

```bash
# Interactive model picker (shows 5 primary models across all providers)
/switch

# Switch to a specific provider (uses its default model)
/provider groq
/provider deepseek

# Switch to a specific model
/model gemini-2.0-flash
/model deepseek-chat
```

## Security

- API keys are stored in plain text in `~/.vega/config.json` — this file is only readable by your user account.
- Keys are **never** transmitted to Raimic Labs servers.
- Vega communicates directly with each provider's official API endpoint.
- To remove a key, run `/connect` and skip the key entry, or manually edit `~/.vega/config.json`.

<Warning>
  Never commit `~/.vega/config.json` to a public repository. Add `~/.vega/` to your global `.gitignore` if needed.
</Warning>

## Troubleshooting Key Issues

**"No API key found" error:**
```bash
# Check your stored keys
/settings
# Re-run the wizard
/connect
```

**Key not working / auth error:**
- Double-check the key was copied completely (no trailing spaces)
- Verify the key is valid in the provider's dashboard
- Make sure you're using the right provider (`/provider nvidia`, etc.)

**"Key looks unusual" warning:**
- Vega checks that your key starts with the expected prefix (`nvapi-`, `AIza`, `gsk_`, `sk-`)
- If your key doesn't match, Vega saves it anyway — some providers change key formats
