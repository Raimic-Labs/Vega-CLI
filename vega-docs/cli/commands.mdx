# CLI Commands

Vega has two types of commands: **shell commands** (run from your terminal before launching Vega) and **slash commands** (typed inside the Vega interactive session).

---

## Shell Commands

These are run directly from your terminal:

| Command | Description |
|---------|-------------|
| `vega` | Start the interactive Vega session |
| `vega "prompt"` | One-shot mode — run a single prompt and exit |
| `vega --version` / `-v` | Show the installed Vega version |
| `vega --help` / `-h` | Show help table and available flags |
| `vega --model <id>` / `-m` | Start with a specific model pre-selected |
| `vega --provider <name>` / `-p` | Start with a specific provider |
| `vega --no-logo` | Skip the ASCII logo on startup |
| `vega build <goal>` | Build a project without entering interactive mode |
| `vega models [provider]` | List all available models (optionally filter by provider) |
| `vega config [key] [value]` | View or set a config value |
| `vega version` | Print version banner |

### Examples

```bash
# Start normally (shows ASCII logo + enters interactive session)
vega

# One-shot: ask a question and exit
vega "What is a closure in JavaScript?"

# Start with a specific model
vega --model deepseek-chat

# Start with a specific provider (uses that provider's default model)
vega --provider groq

# Build a project without interactive mode
vega build "FastAPI todo app with SQLite" --out ./my-todo-api

# List all available models
vega models

# List models for a specific provider
vega models nvidia

# View current config
vega config

# Set a config value
vega config temperature 0.9
```

## In-Session Slash Commands

Once inside the Vega interactive shell, you can use these slash commands at any prompt:

### `/help`  `/h`

Show the full command reference table.

```
/help
```

---

### `/connect`

Launch the provider wizard to add or update an API key. Also hot-swaps the active provider after connecting.

```
/connect
```

```
  Choose a provider:

    1. NVIDIA NIM (free tier)  →  https://build.nvidia.com
    2. Google Gemini           →  https://aistudio.google.com
    3. Groq (free tier)        →  https://console.groq.com
    4. DeepSeek                →  https://platform.deepseek.com
    5. Skip for now (set keys later with /connect)

  Choice [1-5]: 1
  Paste your NVIDIA NIM API key: nvapi-...
  ✔  Connected to NVIDIA NIM (free tier)!
```

---

### `/switch`  `/s`

Interactively pick from the 5 primary curated models across all providers.

```
/switch
```

```
  ✦ Switch Active Model

    1.   Llama 3.1 405B (NVIDIA) (meta/llama-3.1-405b-instruct)
    2.   Gemini 2.0 Flash (Google) (gemini-2.0-flash)
    3. ★ Llama 3.3 70B (Groq) (llama-3.3-70b-versatile)
    4.   DeepSeek-V3 (DeepSeek) (deepseek-chat)
    5.   Kimi K2.6 (NVIDIA) (moonshotai/kimi-k2-instruct)

  Choose a model [1-5]: 5
  ✔  Switched active model to: Kimi K2.6 (NVIDIA)
```

---

### `/model [id]`  `/m [id]`

View or set the active model by ID. If no ID is given, shows the current model.

```
/model                          # Shows current model
/model deepseek-chat            # Switches to DeepSeek-V3
/model gemini-2.0-flash         # Switches to Gemini Flash
/model moonshotai/kimi-k2-instruct
```

---

### `/models`

List all models in the registry with their context windows and notes. Shows a ★ next to the active model.

```
/models
```

---

### `/provider [name]`  `/p [name]`

View the active provider or switch to another one. Switches to that provider's default model automatically.

```
/provider              # Shows current provider
/provider nvidia       # Switches to NVIDIA (default: llama-3.1-405b-instruct)
/provider google       # Switches to Google (default: gemini-2.0-flash)
/provider groq         # Switches to Groq (default: llama-3.3-70b-versatile)
/provider deepseek     # Switches to DeepSeek (default: deepseek-chat)
```

---

### `/agents`

Display the full agent roster with icons, triggers, and descriptions.

```
/agents
```

---

### `/build <goal>`

Launch project builder mode directly. If no goal is provided, Vega will prompt you for one.

```
/build a weather dashboard in React
/build a Python CLI tool for bulk file renaming
/build a REST API with FastAPI and PostgreSQL
```

---

### `/agent <goal>`

Run the autonomous ReAct agent towards a goal. The agent plans, calls tools (write_file, read_file, list_files, run_shell), and iterates until complete.

```
/agent Build a Python web scraper for Hacker News headlines
/agent Create a Makefile with common dev tasks
```

---

### `/clear`  `/cls`

Clear the terminal screen and reset the conversation history (keeps the system prompt).

```
/clear
```

---

### `/history`

Show the last 10 user prompts from saved sessions.

```
/history
```

---

### `/export [path]`

Export the current session to a Markdown file. If no path is given, auto-generates a filename.

```
/export                          # → vega_export_20260622_143022.md
/export ./my-session.md
```

---

### `/settings`

Show all current configuration values in a formatted table.

```
/settings
```

---

### `/config [key] [value]`  `/cfg [key] [value]`

View or set a configuration key. Without arguments, shows all settings.

```
/config                          # Show all config
/config temperature              # Show one key
/config temperature 0.9          # Set a key
/config max_tokens 8192
/config show_token_count false
```

---

### `/system [prompt]`  `/sys [prompt]`

View or set the system prompt. Without a prompt, shows the current system prompt.

```
/system                          # Show current system prompt
/system You are a concise Python expert. Only use Python 3.12+.
```

---

### `/tokens`

Show token usage statistics for the current session.

```
/tokens
# → Turns: 5 | Output tokens: ~2,450 | Context messages: 11
```

---

### `/reset`

Reset all configuration to factory defaults (overwrites `~/.vega/config.json`).

```
/reset
```

---

### `/version`

Print the installed Vega version, tagline, and GitHub URL.

```
/version
```

---

### `/exit`  `/quit`  `/q`

Save the session and exit Vega cleanly.

```
/exit
```

---

## Keyboard Shortcuts

| Shortcut | Behavior |
|----------|----------|
| `Ctrl-D` (EOF) | Save session and exit cleanly |
| `Ctrl-C` | Interrupt current response (mid-stream) or get hint to use `/exit` |

---

## One-Shot Mode

Run a single prompt from the shell without entering interactive mode:

```bash
vega "Explain the difference between async and multithreading in Python"
vega "Write a bash script to backup all .py files in the current directory"
```

The response streams to the terminal and Vega exits after it completes.
