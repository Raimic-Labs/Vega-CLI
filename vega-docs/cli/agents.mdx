# Agents

Vega uses a **multi-agent architecture** — instead of one general-purpose AI doing everything, Vega routes your prompt to a specialized agent optimized for that type of task. This means better code quality, fewer hallucinations, and more relevant output.

---

## What is an Agent?

An agent in Vega is a specialized configuration of an AI model that includes:

- A **preferred model** tuned for a specific task (code generation, planning, debugging, etc.)
- A **trigger classifier** that decides when to activate based on keywords in your prompt
- **Graceful fallback** — if the agent's provider isn't configured, Vega uses your active provider seamlessly

Agents don't change the conversation — they silently route to a better model for the task at hand.

## How Auto-Routing Works

When you type a prompt, Vega runs a lightweight keyword classifier before sending it to an agent:

```
Your prompt
     │
     ▼
┌─────────────────────┐
│   Intent Classifier  │  ← fast keyword matching on your input
└─────────────────────┘
     │
     ├── "build a website" ────────────► ⟡  CodeAgent (Kimi K2)
     ├── "explain what a REST API is" ─► ⚡ FastAgent (Llama 4 Scout)
     ├── "plan the architecture" ──────► 🗺 PlannerAgent (DeepSeek-V3)
     ├── "fix this bug in my code" ────► 🔧 DebugAgent (Kimi K2)
     ├── "review my code for issues" ──► 👁 ReviewAgent (DeepSeek-V3)
     └── anything else ──────────────► Active provider (no agent)
```

The classifier looks at your prompt's **keywords and structure** — not just individual words. It can correctly route:

- "Make me a todo app with React" → **CodeAgent** (understands "make" + "app" = build intent)
- "What's the difference between REST and GraphQL?" → **FastAgent** (understands "what's" = quick Q&A)
- "Design a microservices system for an e-commerce platform" → **PlannerAgent** (understands "design" = architecture)
- "My Python script keeps throwing AttributeError" → **DebugAgent** (understands "error" = debug)

## Agent Fallback Behavior

If the agent's preferred provider **doesn't have an API key configured**, Vega automatically falls back to your active provider:

```
  Routed to CodeAgent (moonshotai/kimi-k2-instruct)
```
→ if NVIDIA key exists, uses Kimi K2 (ideal)

```
  # No NVIDIA key → falls back silently to active provider
```
→ if no NVIDIA key, uses your current model (still works, just not the specialized model)

This means Vega always produces output, even if you've only configured one provider.

## Specialized Agent Prompts

Each agent uses a system prompt tuned for its domain. The **CodeAgent** and **DebugAgent** both use Kimi K2 because it excels at multi-file code generation and complex debugging. The **PlannerAgent** and **ReviewAgent** use DeepSeek-V3 for its strong reasoning at low cost.

## The Autonomous Agent (`/agent`)

Beyond the routing agents above, Vega includes a full **autonomous ReAct agent** you can invoke directly:

```bash
/agent Build a Python web scraper that collects headlines from HackerNews
```

The ReAct agent runs a **multi-step Reason + Act loop**:

1. **Thought** — reasons about what to do next
2. **Action** — calls a tool (write file, read file, list files, run shell command)
3. **Observation** — sees the result
4. **Repeat** — until the goal is complete or max iterations reached

### Available Tools

| Tool         | Description                                           |
|--------------|-------------------------------------------------------|
| `write_file` | Write content to a file path                          |
| `read_file`  | Read the contents of a file                           |
| `list_files` | List files in a directory                             |
| `run_shell`  | Execute a shell command (warns before running)        |

### Example Session

```
  ✦ › /agent Build a FastAPI hello-world app

  ──────────── Vega Agent — Build a FastAPI hello-world app ────────────
  Iteration 1/10
  Thought: I need to create a FastAPI app with a hello world endpoint.
  Action: write_file
  Args: {"path": "main.py", "content": "from fastapi import FastAPI\n..."}
  Tool [write_file] → Written: ./main.py

  Iteration 2/10
  Thought: I should also create requirements.txt.
  Action: write_file
  Args: {"path": "requirements.txt", "content": "fastapi\nuvicorn\n"}
  Tool [write_file] → Written: ./requirements.txt

  Action: FINISH
  Answer: Created main.py and requirements.txt. Run with: uvicorn main:app --reload
  ✔  Agent finished.
```

## Manually Overriding the Agent

You can force a specific model for any conversation using `/switch` or `/model`:

```bash
# Interactively pick from the 5 primary models
/switch

# Set a specific model by ID
/model deepseek-chat
```

This overrides the agent routing for the rest of the session.
