### тЬи New Features

#### Multi-Agent Architecture
Vega launches with 6 specialized AI agents, each tuned for a different development domain:

- ЁЯМР **WebAgent** тАФ HTML/CSS/JS websites and landing pages
- ЁЯУ▒ **AppAgent** тАФ React and Vue frontend applications
- ЁЯРН **CodeAgent** тАФ Python scripts, algorithms, and CLI tools
- тЪЩя╕П **BackendAgent** тАФ REST APIs, servers, and database integrations
- ЁЯУК **DataAgent** тАФ Data analysis, Pandas, and chart generation
- ЁЯдЦ **GeneralAgent** тАФ General questions, writing, and planning

#### Automatic Intent Routing
Vega analyzes your prompt and automatically selects the most appropriate agent. No manual configuration needed тАФ just describe what you want.

#### Multi-Model Support
Connect API keys from 7 different providers and switch between 14+ models:

| Provider | Models |
|----------|--------|
| Google | `gemini-2.5-pro`, `gemini-2.0-flash`, `gemini-1.5-pro` |
| OpenAI | `gpt-4o`, `gpt-4o-mini` |
| Anthropic | `claude-3-5-sonnet`, `claude-3-haiku` |
| Groq | `llama-3.3-70b-versatile`, `llama-3.1-8b-instant`, `mixtral-8x7b` |
| DeepSeek | `deepseek-chat`, `deepseek-coder` |
| Mistral | `mistral-large-latest` |
| NVIDIA NIM | `nvidia/llama-3.1-nemotron-70b` |

#### Interactive Slash Commands
The following slash commands are available inside the Vega session:

- `/connect` тАФ Add or update API keys
- `/model` тАФ Switch AI model mid-session
- `/agent` тАФ Manually override agent routing
- `/agents` тАФ List all available agents
- `/clear` тАФ Clear screen and reset conversation history
- `/save` тАФ Save session to a file
- `/history` тАФ Show message history
- `/status` тАФ Display current model, agent, and key status
- `/help` тАФ Show all available commands

#### Direct File Output
All generated code is written directly to the current working directory. No clipboard required тАФ just describe what you want and the files appear in your project folder.

#### Persistent API Key Storage
API keys are saved locally to `~/.vega/config.json` and persist across sessions. Add once, use forever.

#### Session Context
Vega maintains conversation context throughout a session, allowing you to refine, iterate, and build incrementally.

### ЁЯРЫ Known Issues

- **Long responses may truncate** on models with strict output limits. Workaround: ask Vega to "continue" or break into smaller requests.
- **Existing file editing** is not yet supported тАФ Vega generates new files only. Paste existing code into the prompt for context.
- **Windows Command Prompt** has limited Unicode rendering. Use Windows Terminal for the best experience.
- **Ollama/local model support** is not yet available. Planned for v0.2.0.
