# Building APIs with Vega

Vega's **BackendAgent** specializes in generating server-side code вЂ” REST APIs, authentication systems, database schemas, and microservices. It produces clean, maintainable code with proper error handling, input validation, and documented endpoints.

## Example 1 вЂ” Flask REST API with Auth

**Prompt:**
```
Build a Flask REST API for a task management app. Include user registration and login 
with JWT tokens, and CRUD endpoints for tasks (each task has title, description, status, 
due_date, priority). Use SQLite with SQLAlchemy. Protect task endpoints with JWT auth. 
Include proper error handling and HTTP status codes.
```

**Agent:** вљ™пёЏ BackendAgent  
**Stack:** Python + Flask + SQLAlchemy + SQLite

**What gets created:**
```
./task-api-flask/
в”њв”Ђв”Ђ app.py
в”њв”Ђв”Ђ models/
в”‚   в”њв”Ђв”Ђ user.py
в”‚   в””в”Ђв”Ђ task.py
в”њв”Ђв”Ђ routes/
в”‚   в”њв”Ђв”Ђ auth.py
в”‚   в””в”Ђв”Ђ tasks.py
в”њв”Ђв”Ђ middleware/
в”‚   в””в”Ђв”Ђ auth.py
в”њв”Ђв”Ђ requirements.txt
в””в”Ђв”Ђ README.md
```

**Generated endpoints:**
```
POST   /auth/register     вЂ” Register a new user
POST   /auth/login        вЂ” Login and receive JWT token
GET    /tasks             вЂ” List all user's tasks
POST   /tasks             вЂ” Create a new task
GET    /tasks/:id         вЂ” Get a specific task
PUT    /tasks/:id         вЂ” Update a task
DELETE /tasks/:id         вЂ” Delete a task
```

**Usage:**
```bash
pip install -r requirements.txt
python app.py

# Register
curl -X POST http://localhost:5000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "secret123"}'

# Login
curl -X POST http://localhost:5000/auth/login \
  -d '{"email": "user@example.com", "password": "secret123"}'

# Returns: {"token": "eyJhbGciOiJIUzI1..."}

# Create task (with token)
curl -X POST http://localhost:5000/tasks \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1..." \
  -H "Content-Type: application/json" \
  -d '{"title": "Launch MVP", "priority": "high", "due_date": "2025-03-01"}'
```

## Testing Your Generated APIs

Vega often suggests testing commands in the README. You can also ask for tests:

```
You: Add pytest tests for all the auth endpoints in the Flask API
```

```
You: Generate a Postman collection JSON file for all the endpoints
```

```
You: Write integration tests for the Express API using Jest and Supertest
```

<Tip>
  **Mention security requirements.** If you need rate limiting, CORS configuration, SQL injection prevention, or request validation, say so explicitly. The BackendAgent will include these if asked.
</Tip>

<Tip>
  **Ask for `.env.example`.** Always ask Vega to generate a `.env.example` file so you know exactly which environment variables to configure.
</Tip>
